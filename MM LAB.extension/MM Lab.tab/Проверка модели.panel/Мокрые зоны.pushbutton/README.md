# Мокрые зоны — проверка помещений

## Описание

Плагин проверяет все помещения в проекте на пересечение с проекцией мокрых помещений,
расположенных на уровне выше. Помещения, попадающие под мокрую зону сверху,
выводятся в табличный отчёт.

## Логика работы

1. Пользователь вводит через запятую названия (или части названий) мокрых помещений.
2. Пользователь выбирает стадию проекта из выпадающего списка.
3. Плагин собирает все размещённые помещения в заданной стадии.
4. Составляется список уровней, на которых есть хотя бы одно помещение,
   и сортируется по высотной отметке.
5. Для каждого помещения определяется «уровень выше» — следующий уровень
   в отсортированном списке.
6. На верхнем уровне находятся мокрые помещения (по совпадению имени с фильтром).
7. Контуры мокрых помещений проецируются на плоскость Z=0.
8. Контур проверяемого помещения также проецируется на Z=0.
9. Выполняется Boolean Intersect двух экструзий — если объём пересечения > 0,
   помещение считается попадающим под мокрую зону.
10. Результат выводится таблицей: Id, номер, имя помещения.

## Параметры помещений

Используются системные параметры Revit:

| Параметр | Назначение |
|----------|------------|
| `ROOM_NAME` | Имя помещения (для фильтрации мокрых зон) |
| `ROOM_NUMBER` | Номер помещения (для отчёта) |
| `ROOM_PHASE` | Стадия помещения (для фильтрации по стадии) |

## Зависимости

Внешних зависимостей нет. Используются только стандартные модули Revit API и PyRevit.

## Совместимость

- Revit 2020
- Revit 2022
- Revit 2024

Все вызовы Revit API совместимы со всеми тремя версиями. Версионно-зависимых API
(UnitTypeId, SpecTypeId и т.д.) скрипт не использует.

## Интерфейс

При запуске открывается диалоговое окно с двумя полями:

- **Названия мокрых помещений** — текстовое поле, через запятую. По умолчанию:
  `санузел, ванная, душ, туалет, постирочная, прачечная`
- **Стадия проекта** — выпадающий список стадий проекта.

Результат отображается в окне вывода PyRevit в виде markdown-таблицы.
